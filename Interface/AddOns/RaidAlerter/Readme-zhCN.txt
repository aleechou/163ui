RaidAlerter团队报警系统

插件语言：简体中文、繁体中文
插件作者：原作者Oldhand
内存占用：初始内存占用约300K，超限会自动清理
资源占用：视战斗状况而定，在大规模战役中将占用较多的CPU资源。不需要的报警功能建议及时关闭。
插件简介：提供RAID和5人队伍的各种特色警报。

RaidAlerter的前身是RaidCompere，因RaidCompere占用资源过大，早已停止更新。后原作者Oldhand将RaidCompere功能做了大量精简，成为最初的RaidAlerter。
现在RaidAlerter主要提供MT血量危急、嘲讽抵抗、OT、驱散、猎人宁神、被控制、打破控制、团队检查和5人小队警报等，这些绝大部分都是首领模块所不具备的功能。

【插件功能】
带"★"的功能需预先设置MT，否则无效。
◇血量监视：
　・团队状态下MT的血量危急警报★；MT阵亡通报★
　・5人小队状态下队友的血量危急警报；通报自己死亡
　・Boss血量通报
◇仇恨报警：
　・Boss战时警报仇恨超过MT者，防OT于未然。
　・对于比较危险的怪物（包括Boss和一些小怪），警报第一仇恨者。非TANK职业获得第一仇恨通常就是OT。团队状态宜先设定好MT。
◇技能通报：
　・TANK职业的嘲讽技能
　・队友被控制
　・打破控制技能：如开羊等的通报
　・驱散魔法
　・打断技能
　・战士：盾墙、破釜沉舟、法术反射
　・猎人：标记、误导、宁神
　・盗贼：嫁祸诀窍
　・术士：灵魂石
　・圣骑：神圣干涉、圣佑术
　・法师：变形术、法术反制、法术偷取
　・德鲁伊：生存本能
　・死亡骑士：冰封之韧、反魔法护罩、反魔法领域、吸血鬼之血
　・召唤团队物件的法术技能
◇阵亡通报：战斗结束后若有人阵亡，则
　・通报阵亡人数，同时视情况通报和法力和灵魂绑定状态，便于及时复活和RL决定是否稍息。
　・在本机用白字显示最先进入战斗状态者（不会影响他人频道）。由于受诸多因素影响，此信息仅供参考。
◇特定Boss警报：
　・NEF：堕落治疗、狂野变形
　・TAQ：双子战监视：提供AOE技能打到小虫的警报
　（上面两个功能现在已基本用不上...但说不定哪天心血来潮会去旧地重游...故而保留之）
◇生命危急自动喊话：当玩家自身血量过低（默认30%）时自动喊话请求治疗。需属于团队模式设置的正常队员。
　此功能进游戏时均默认为关闭，退出游戏后不保存设置。请根据需要使用状态条快捷键或者命令/ral myhp开关。
◇仇恨提示：在屏幕上显示你的仇恨信息。
　・当你获得某个怪物的第一仇恨时
　・当目标的目标转向你时
　・当你仇恨过高时
◇团队检查：可检查并公布团队的到位、Buff、灵魂石绑定、合剂和药剂效果等状态。
◇团队状态条
　・实时显示3个重要参数：团队法力、治疗法力和死亡人数
　・在悬停提示中显示团队PvP、到位、灵魂石、药剂状态等。

【使用说明】
◇状态条操作：
　　左键点击：开关警报
　　右键点击：打开选项菜单
　　右键拖动：调整团队状态条位置
　　Ctrl+左键：公布到位检查结果
　　Ctrl+右键：公布合剂检查结果
　　Alt+左键：MT目标锁定检查
　　Alt+右键：公布增强药剂检查结果
　　Shift+左键：检查团队Buff
　　Shift+右键：立即清理插件内存
　　Ctrl+Alt+左键：血量危急自动喊话开关

◇发送团队警报需要团长L或者助理A权限，否则只在本机显示。多人启用RaidAlerter时，会自动选择一玩家作为报警发送方。若无需警报请及时关闭，可减少资源占用。
◇团队模式人数：比如，设定为25人团队模式，那么认为1-5队属于正常队员，其他则被判定为替补。替补成员不影响警报和实时数据。
◇内存清理界限，默认416K，增大此设置可以减少内存自动清理频率。个人建议设定为350～500KB。
◇命令帮助：/raidalerter 或 /ral help

建议配合使用的插件：团队助手（oRA2、oRA3或CT_RaidAssist） + 首领模块（DBM或BigWigs）

虽然所有的功能设定均有其用意，但众口难调，有些细节如果不合你意的话你可以DIY――用文本编辑工具（如Editplus或UltraEdit）打开RaidAlerter.lua文件：
比如警报方式
有的玩家想用喊话、说等不同的警报方式，找到比如：
	RAL.SendMSG(string.format(RAL_TEXT_ALERT_19,srcName,dstName), AlertChan);
可以把最后面的AlertChan改成（引号必须）：
　　　　"RAID" - 团队
　　　　"RAID_WARNING" - 团队通知
　　　　"PARTY" - 小队
　　　　"SAY" - 说
　　　　"YELL" - 喊
　　　　"MYCHAN" - 仅在本机讯息栏显示
　　　　"MYSCR" - 仅在本机屏幕显示

最后，关于插件语言：目前只支持中文。
要支持其他语言，并不是简单的翻译一下就能用，而是要把所有名词和格式做到精准。
玩家可自行尝试修改。

BTW：UI源码本来就是完全开放的，任何人都可以在尊重作者的前提下自己进行修改。
